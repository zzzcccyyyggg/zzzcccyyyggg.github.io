快速模幂算法

数学原理

流程如下：

  1. 将指数 b 转换成二进制形式，记为 b = b_k b_{k-1} ... b_2 b_1 b_0。
  2. 初始值设为 x = 1，然后对于每一位 b_i，如果 b_i = 1，则将 x等于x乘以 \(a^{2i}\) mod m 。
  3. 最后返回 x mod m。

由于在每一步中都只需要进行一次乘法和取模操作，因此快速模幂的时间复杂度为 O(log b)。

中国剩余定理

算法原理

中国剩余定理是一种求解同余方程组的方法。同余方程组形如：

\[x \equiv a_1 \pmod{m_1}\\x \equiv a_2 \pmod{m_2}\\\vdots\\x \equiv a_k \pmod{m_k}\]

其中，\(a_1, a_2, \dots, a_k\) 和 \(m_1, m_2, \dots, m_k\) 都是已知的整数。

设 \(M = m_1 m_2 \dots m_k\)，则 \(M/m_i\) 和 \(m_i\) 互质。记 \(M_i = M/m_i\)，则存在整数 \(y_i\) 使得 \(M_i y_i \equiv 1 \pmod{m_i}\)。则 \(x = \sum_{i=1}^{k} a_i M_i y_i\) 是同余方程组的解，且 \(x \equiv r \pmod{M}\)，其中 \(r = \sum_{i=1}^{k} a_i M_i y_i\)。

MillerRobin素性检测算法

Miller-Rabin 检测算法是一种用于判断一个数是否为质数的算法，它在计算复杂度和可靠性之间取得了一个很好的平衡。

算法原理

Miller-Rabin 算法的基本思路是，对于一个待判断的整数 n，如果 n 是质数，则存在一个整数 a（1 < a < n），使得：

​ \[ a^{n-1}\equiv1modn \]

此外，对于一个合数n，如果能够找到一个整数a，满足：

​ \[a^{n-1}\neq1modn\]

那么n一定不是质数，并且由于在小于n的整数中米勒拉宾证据出现的概率是75%因此，Miller-Rabin 算法的核心是选择一个适当的 a，利用快速幂算法计算 \(a^{n-1} \bmod n\)，判断其是否等于 1，如果不等于 1，则可以确定 n 不是质数。为了提高算法的可靠性，Miller-Rabin 算法需要进行多次检测，每次选择不同的随机数 a，如果多次检测都得到了 \(a^{n-1} \equiv 1 \pmod{n}\)，那么可以认为 n 是质数的概率非常高。

实验过程

伪代码

以下是 Miller-Rabin 素性检测算法的伪代码：

输入：正整数 \(n\) 和检测次数 \(k\)

输出：True 表示 \(n\) 可能为素数，False 表示 \(n\) 不是素数

  1. 如果 \(n < 2\)，返回 False
  2. 如果 \(n = 2\)，返回 True
  3. 如果 \(n\) 是偶数，返回 False
  4. 将 \(n - 1\) 表示成 \((2^r) \cdot d\) 的形式，其中 \(d\) 是奇数
  5. 进行 \(k\) 次检测，每次检测如下：
    1. 随机选择一个整数 \(a\)，\(2 \leq a \leq n-2\)
    2. 计算 \(x = a^d \bmod n\)
    3. 如果 \(x = 1\) 或者 \(x = n-1\)，则跳过下一步
    4. 对 \(r-1\) 次幂运算，如果 \(x = n-1\)，则跳出循环
    5. 如果所有的幂运算都不满足 \(x = n-1\)，则返回 False
  6. 如果经过 \(k\) 次检测都没有发现 \(n\) 不是素数，返回 True

其中，\(\bmod\) 符号表示模运算，\(\cdot\) 符号表示乘法运算，\(2 \leq a \leq n-2\) 表示 \(a\) 的取值范围，\(k\) 表示进行检测的次数，\(r\) 和 \(d\) 表示将 \(n-1\) 表示成 \((2^r) \cdot d\) 的形式中的两个参数。

厄拉多塞筛法

算法原理

厄拉多塞筛法是一种用于求解素数的算法。其基本思想是从2开始，将每个质数的倍数都标记成合数，以达到筛选素数的目的。

算法流程如下：

  1. 构造一个从2开始的自然数序列。
  2. 取序列中当前未被标记为合数的最小素数p，将其作为新的质数。
  3. 将p的倍数，即p2、p3、p*4……标记为合数。
  4. 如果序列中未被标记为合数的最小素数大于p，说明还有未被筛选的素数，返回步骤2；否则算法结束。

有限域四则运算

算法流程

首先定义五个函数：

  • addGF(a, b)函数实现了在二元域上的加法，使用异或运算实现；
  • subGF(a, b)函数实现了在二元域上的减法，使用异或运算实现；
  • mulGF(a, b, n)函数实现了在二元域上的乘法，使用移位和异或运算实现，其中参数n表示模多项式，即二元域上的特征多项式；
  • divGF(a, b)函数实现了在二元域上的除法，使用移位和异或运算实现，其中返回值为二元组(c1, c2)，c1表示商，c2表示余数；
  • operationGF(a, b, g)函数根据输入的操作符g，调用相应的函数实现二元域上的加减乘除操作，其中a, b分别表示两个操作数，g为操作符。

接着在主程序中，首先读入三个参数a, g, b，其中a和b为16进制数，g为操作符，然后将a和b转换为整数类型，最后调用operationGF(a, b, g)函数执行对应的操作。

二元域上的加减法运算很容易理解，即对应位进行异或运算即可。乘法的实现主要是使用移位和异或运算，不断将a左移一位并模n取余，若b当前的二进制位为1，则将当前的a与result进行异或操作；而除法的实现主要是使用移位和异或运算，通过不断将b左移，然后异或操作a来得到商和余数。值得注意的是，除法的实现需要先判断a和b的位数关系，如果a的位数小于b，则商为0，余数为a。而如果a的位数与b的位数相等，则商为1，余数为a异或b。

有限域的逆元的生成

算法原理

假设 \(x\) 和 \(n\) 是正整数，若存在正整数 \(a\)，使得 \(ax \equiv 1 \pmod{n}\)，则称 \(a\) 是 \(x\) 在模 \(n\) 意义下的逆元。

在求解逆元时，使用有限域扩展欧几里得算法。假设要求 \(x\) 在模 \(n\) 意义下的逆元 \(a\)，则需要解方程 \(ax+ny=1\)，其中 \(x\) 和 \(y\) 是扩展欧几里得算法的输出结果。如果 \(x<0\)，则可以将 \(x\) 加上 \(n\) 直到 \(x\) 变成非负整数。

若 \(x\) 在模 \(n\) 意义下无逆元，则 \(x\) 和 \(n\) 不是互质的，即它们有一个大于 \(1\) 的公约数，则此时无逆元。

本原多项式的判定与生成

算法原理

若 \(f(x)\) 是 \(GF(2^2)\) 上的不可约多项式，且 \(f(x)\) 的根是 \(GF(2^8)\) 的本原元，则称 \(f(x)\) 为 \(GF(2^2)\) 上的本原多项式；另一等价定义如下：

若一个 \(n\) 次多项式 \(f(x)\) 满足以下条件：

  1. \(f(x)\) 为不可约多项式；
  2. \(f(x)\) 可整除 \(x^{2n}+1\) (\(n=2^m-1\))；
  3. \(f(x)\) 不能整除 \(x^q+1\) (\(q<n\))；

则称 \(f(x)\) 为 \(n\) 次本原多项式。

故我们只需对遍历0b100000000与0b1000000000中的所有数字，并利用这三个条件进行判定找出所有不可约多项式即可

中国剩余定理可以改进的地方

  1. 线性同余方程组的情况：对于一些线性同余方程组，可以使用高斯消元法来求解，比中国剩余定理更为高效。

  2. 不同质数情况下的方程组：

    在不同质数情况下的方程组中，中国剩余定理（CRT）可以使用 Garner 算法或 CRT 重构算法来加速。这是因为当模数不同时，CRT 不能直接使用，需要将方程组转化为同余方程组。下面分别介绍 Garner 算法和 CRT 重构算法。

    Garner 算法是一种基于连续求模的快速求解同余方程组的方法。具体来说，它将同余方程组中的每个方程分解为某个数乘以模数加上一个余数的形式，然后依次计算每个数。该算法具有良好的时间复杂度和空间复杂度，可以处理任意多个不同质数的同余方程组，但需要对每个方程进行分解，对大质数的处理会有一定的计算量。

    CRT 重构算法也是一种快速求解同余方程组的方法。该算法的核心思想是将不同质数的模数两两合并，然后使用扩展欧几里得算法求解合并后的方程组，最后利用中国剩余定理将合并后的解还原为原方程组的解。该算法不需要对每个方程进行分解，因此不受大质数的影响，但需要多次使用扩展欧几里得算法，对时间和空间的要求较高。

    总体而言，Garner 算法适用于任意多个不同质数的同余方程组，且在质数较小时效率更高，而 CRT 重构算法适用于质数数目较少的情况，但是计算时间和空间复杂度都比 Garner 算法高一些。

多项式牛顿迭代法

经过查阅资料，或许我们可以使用牛顿迭代法

牛顿迭代法（Newton's method），也称作牛顿-拉夫逊迭代法（Newton-Raphson method），是一种用于寻找方程的根的迭代方法。

牛顿迭代法的基本思想是：假设我们已经有了一个初始估计值 \(x_0\)，然后通过下面的公式不断迭代，直到收敛：

\[x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}\]

其中 \(f(x)\) 代表方程，\(f'(x)\) 代表 \(f(x)\) 的导数。\(x_{n+1}\) 是 \(x_n\) 的一个更好的近似值。

简单来说，牛顿迭代法的思路是：假设 \(f(x)\) 在 \(x_0\) 处有一根，那么可以将其近似为其一阶泰勒展开式，然后用一阶泰勒展开式的根作为 \(f(x)\) 的根的近似值，即：

\[f(x) \approx f(x_0) + f'(x_0)(x - x_0)\]

将上式的左边看做0，解得：

\[x \approx x_0 - \frac{f(x_0)}{f'(x_0)}\]

接着，用新的近似值 \(x_1\) 代替 \(x_0\)，然后重复上述步骤，直到达到精度要求为止。

牛顿迭代法的优点在于收敛速度非常快，通常情况下可以在几次迭代之后达到指定的精度要求。但是它也有一些缺点，最主要的是如果方程的初始估计值不够好，可能会导致迭代发散，或者迭代过程中出现震荡的情况。

算法流程

  5. 在解密函数affine_decode中，先根据a计算a的逆元a_inv，然后根据a_inv和b将密文恢复为明文，最后将明文拼接起来并返回。
  6. 最后打印result即可

思考与讨论

可以看到，对于明文中的每个字符，它们都被加上了密钥中对应位置的字符，然后对 26 取模得到了密文中的每个字符。

解密的过程与加密过程相似，只需要使用密钥的字符对应的数字减去密文字符对应的数字，然后再对 26 取模，得到明文字符对应的数字。最后，将这些数字转换回字母就可以得到明文了。

维吉尼亚密码的优点在于使用了密钥对明文进行多次代换，从而增加了密码的安全性。缺点在于如果密钥过短，或者密文长度足够长，那么密码就容易被破解

栅栏密码

代码流程

  1. 输入三个参数：s、k 和 op。
  2. 检查 op 的值以确定是加密还是解密消息。
  3. 如果 op 为 1，则加密s。 创建一个名为 fence 的空列表列表，其中包含 k 个空子列表。 将名为 count 的计数器变量初始化为 0。
  4. 构造循环，遍历消息字符串中的每个字符， 将字符添加到 fence 的第 count 个子列表中，然后将 count 加 1 并对 k 取模以确保 count 保持在子列表索引范围内。
  5. 一旦所有字符都添加到适当的子列表中，按顺序连接子列表以形成称为 cipher 的加密消息字符串。
  6. 将密码字符串作为加密消息返回。
  7. 如果 op 为 0，则解密s。 创建一个名为 fence 的空列表列表，其中包含 k 个空子列表。
  8. 用消息字符串的长度除以k，计算fence的每个子列表中的字符数（无法整除的话不同子列表之间的字符数不相同，部分为div，部分为div+1）。 将结果存储在 div 中。 通过将消息长度与 k 取模来计算具有附加字符的子列表的数量。 将结果存储在 mod 中。 将名为 temp 的临时变量初始化为 0。
  9. 如果 mod 不为 0，则存在将具有附加字符的 mod 子列表。 循环遍历 mod 的范围。 对于每个子列表索引 i，将 message 中的下一个 div+1 个字符分配给 fence[i] 并将 temp 递增 div+1 以移动到下一组字符。
  10. 一旦分配了所有带有附加字符的子列表，循环遍历 mod 到 k 的范围。 对于每个子列表索引 i，将下一个 div 字符从 message 分配给 fence[i] 并将 temp 递增 div 以移动到下一组字符。
  11. 如果 mod 为 0，则所有子列表的字符数相同。 遍历 k 的范围。 对于每个子列表索引 i，将下一个 div 字符从 message 分配给 fence[i] 并将 temp 递增 div 以移动到下一组字符。
  12. 初始化一个名为 plain 的空字符串来存储解密后的消息。 将名为 length 的变量初始化为 -1。 遍历消息字符串中的每个字符。 通过对当前索引与k取模计算字符所属的子列表的索引。 如果索引为 0，则将长度增加 1 以移动到每个子列表中的下一个字符。 将 fence 子列表中的适当字符附加到 plain。
  13. 返回纯字符串作为解密消息。
  14. 最后将结果打印出来

在写这个程序的过程中，我对python独有的列表推导式有了更加深刻的认识，我发现它真是非常的简洁美观方便！

例：fence = [[] for _ in range(rails)]

这行代码使用了列表推导式的语法，生成了一个由rails个空列表组成的列表。这里的for _ in range(rails)是一个for循环语句，它在range(rails)中迭代rails次，每次迭代都会执行一次循环体。_是一个特殊的变量名，它表示我们不需要使用循环计数器的值，所以将其省略。由于循环体没有使用计数器的值，因此可以将其替换为一个下划线_，表示占位符。循环体中的[]表示创建一个空列表，这个空列表会被添加到最外层的列表中。因此，通过[[] for _ in range(rails)]这一列表推导式，可以创建一个由rails个空列表组成的列表。

伪代码

  1. 定义函数 hill_cipher 用于实现Hill密码算法的加解密过程。
    1. 读入矩阵大小n,读入n * n的整数矩阵key
    2. 读入待加密或解密的字符串s。

    4. 读入操作s数op。
    5. 如果op为1，表示进行加密操作：
      1. 再mod26下计算密文矩阵c_matrix = s_matrix * key。
      2. 调用matrix_to_string将密文矩阵转化为字符串c。
      3. 输出字符串c。
    6. 如果op为2，表示进行解密操作：
      1. 计算key在模26下的逆矩阵inverse_key。
      2. 如果inverse_key不存在，则返回None。
      3. 如果存在，计算明文矩阵p_matrix = s_matrix * inverse_key。
      4. 调用matrix_to_string将明文矩阵转化为字符串c。
      5. 输出字符串p。

讨论与思考

经过资料搜索，发现高斯消元法和通过伴随矩阵求逆的代码实现复杂度都大于等于O(\(N^3\))，并并适用于大型矩阵，而大型矩阵通常采用LU分解法和QR分解法，且由于暂时没想好高斯消元法处理对角线为零的情况如何处理，故采用伴随矩阵求逆的方法。

LU分解法将一个矩阵分解为一个下三角矩阵L和一个上三角矩阵U的乘积，即A = LU。求解逆矩阵时，首先对矩阵进行LU分解，然后求解两个方程组Ly = I和Ux = y，其中I为单位矩阵，y和x分别为两个未知向量。这样可以较为高效地求解矩阵的逆。

QR分解法将一个矩阵分解为一个正交矩阵Q和一个上三角矩阵R的乘积，即A = QR。求解逆矩阵时，首先对矩阵进行QR分解，然后求解方程组Rx = Q-1，其中Q-1为Q的逆矩阵，x为未知向量。这样也可以较为高效地求解矩阵的逆。

这些方法都可以在Python中实现，但实现过程可能比较复杂，需要一定的线性代数基础。在实际应用中，也可以使用一些现成的线性代数库来实现矩阵逆的计算。

单表代替密码的字母频率攻击

对于自然语言，如果取一本非专业书籍，统计足够长的课文就会发现，字母(或字符)出现的频率会反映出相应语言的统计特性。统计大量的课文定会发现，相应语言中每个字母在相应语言中出现的概率。于是便得到该语言字母表上的一个概率分布。

一、英文字母

E T A O I N S H R D L C U M W F G Y P B V K J X Q Z

在上表中，不少字母出现的概率近乎相等。为了应用方便，常将英文字母表按字母出现的概率大小分类，分类情况见下表：

-------------------------- 极高频 E 次高频 T A O I N S H R 中等频 D L 低频 C U M W F G Y P B 甚低频 V K J X Q Z -------------------------- 其它语言和数据也有类似于英语语言的单字母统计特性。如果我们随意统计一段足够长的英文课文，只要内容不是太特殊，其结果一定和上表基本相同。这表明英文的一篇文章中各个字母出现的概率是基本可预测的，它将为密码分析提供一个方面的依据。 语言的单字母统计特性至少在以下两个方面没有反映出英文语言的特征： ⑴根据英文的单字母统计特性可以计算出双字母QE出现的概率为p(QE)=0.00095×0.12702≈1.21×10^(-4) 这就是说，在10^6个双字母的抽样中，QE出现的次数大约应为121次，但这不符合英文课文的实际。因为在英文课文中，QE根本不出现。 ⑵四字母SEND和SEDN在单字母统计特性下出现的概率相等，这也不符合英文的实际。总之，自然语言的单字母统计特性只反映了单字母出现的概率，而没有反映该种语言文字的字母间的相关关系。为了体现自然语言的双字母统计特性，我们需要考察该语言的文字中相邻字母对出现的频数。
