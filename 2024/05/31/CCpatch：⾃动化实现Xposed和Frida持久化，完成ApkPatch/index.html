参考chat给出的启动顺序

  1. 系统启动 Launcher 应用：
    • 用户在设备上启动应用时，系统会首先启动 Launcher 应用（即设备的主界面）。
  2. Launcher 启动应用：
    • 用户点击应用图标后，Launcher 应用会通过 startActivity 请求启动目标应用的主 Activity。
  3. 系统进程创建应用进程：
    • 如果目标应用的进程尚未启动，系统会在 Zygote 进程的基础上创建新的应用进程。
    • Zygote 是 Android 中启动应用进程的母体进程，负责 fork 新的应用进程。
  4. 加载应用的 APK：
    • 新的应用进程启动后，系统会加载应用的 APK 文件。
    • 这包括加载应用的代码（DEX 文件）、资源（如图片、字符串）、本地库（.so 文件）等。
  5. 创建 Application 实例：
    • 系统会创建应用的 Application 类的实例，并调用 onCreate 方法。
    • 在这一步，应用可以执行全局的初始化逻辑。
  6. 创建组件（如 Activity）：
    • 在启动 Activity 时，系统会首先查找应用的 AppComponentFactory（如果有）。
    • 系统会调用 AppComponentFactory 的 instantiateActivity 方法来创建 Activity 实例。
    • 如果没有自定义的 AppComponentFactory，系统会使用默认的实现。
