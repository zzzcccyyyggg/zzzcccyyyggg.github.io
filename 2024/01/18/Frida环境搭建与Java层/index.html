Java反射机制

定义

JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意方法和属性；这种动态获取信息以及动态调用对象方法的功能称为java语言的反射机制。

  • 一个加载的类在JVM中只会有一个Class实例

  • 每个类的实例都会记得自己是由哪个Class实例所生成
  • 通过Class可以完整地得到一个类中的所有被加载的结构

Java.enumerateLoadedClassesSync() 使用案例

这是 Java.enumerateLoadedClasses(callbacks) 的同步版本。它会返回一个包含所有已加载类的数组，而不是使用回调。

这是 Java.enumerateClassLoaders(callbacks) 的同步版本，直接返回一个包含所有类加载器的数组。

Java.enumerateMethods(query)

查询条件 query 是一个特定格式的字符串，用于指定要搜索的类和方法，同时还可以指定匹配的选项。

  • 基本查询格式：查询通常有 "class!method" 的格式，其中 class 是要搜索的类名，method 是要搜索的方法名。使用感叹号 ! 分隔类名和方法名。
  • 通配符：可以使用星号 * 作为通配符。
  • 查询修饰符：查询字符串可以附加一个或多个修饰符，通过在查询字符串后面加上 / 和修饰符来实现：
    • i：进行不区分大小写的匹配。例如，"java.lang.String!startsWith/i" 会匹配 java.lang.String 类中所有以 startswith 为名（不区分大小写）的方法。
    • s：包括方法签名。这意味着你不仅能搜索方法名，还能指定要搜索的方法参数和返回类型。例如，"java.lang.String!indexOf/s" 可以返回 indexOf 方法及其所有重载版本的签名，如 indexOf(int): int。
    • u：仅匹配用户定义的类，忽略系统类。这可以帮助过滤掉 Java 标准库中的类和其他预加载的系统类，仅关注由应用程序定义或者加载的类
